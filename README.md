# Day2-FileHandling CI/CD Demo

##  Workflow Name
**Day2-FileHandling**

## 🛠️ Steps in Workflow
1. Checkout repository  
2. Create a file (`demo.txt`)  
3. Write text into the file (`echo "Hello CI/CD automation!"`)  
4. Read the file contents (`cat demo.txt`)  
5. Upload the file as an artifact  

##  Purpose
This project demonstrates CI/CD automation basics for fresher DevOps roles using GitHub Actions.  
It shows how pipelines can automate file handling tasks in a Linux environment and store outputs as artifacts.

##  How to Run
- Push changes to the `main` branch.  
- GitHub Actions will automatically trigger the workflow.  
- Check the **Actions tab** to see logs and download the uploaded artifact (`demo.txt`).
